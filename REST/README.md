# Go REST API 프레임워크 비교

4가지 인기 있는 Go 웹 프레임워크로 구현한 동일한 REST API 서버입니다.

## 📁 프로젝트 구조

```
REST/
├── Gin/          # 고성능 마이크로 프레임워크
├── Chi/          # 경량 표준 호환 라우터
├── Echo/         # 기능 풍부한 균형잡힌 프레임워크
├── Beego/        # 풀스택 MVC 프레임워크
└── README.md     # 이 파일
```

## 🎯 각 프레임워크 특징

### 1. Gin - 고성능 & 인기 최고

```bash
cd Gin && go run main.go
```

**장점**:
- ⚡ 최고 성능 (50,000+ req/s)
- ⭐ 가장 큰 커뮤니티 (77k stars)
- 🎯 자동 JSON 바인딩 및 검증
- 📦 풍부한 미들웨어
- 🚀 빠른 개발

**단점**:
- ❌ 표준 http 인터페이스가 아님
- ❌ 의존성 많음 (28개)

**추천 사용처**:
- 스타트업 MVP/프로토타입
- 높은 성능이 필요한 서비스
- 빠른 개발이 중요한 경우

---

### 2. Chi - 경량 & 표준 호환

```bash
cd Chi && go run main.go
```

**장점**:
- 🎯 표준 `net/http` 호환
- 🪶 가장 적은 의존성 (1개)
- 📐 명확한 중첩 라우팅
- 🔧 완전한 제어
- 🏗️ 마이크로서비스 친화적

**단점**:
- ❌ 자동 바인딩 없음
- ❌ 수동 검증 필요
- ❌ 보일러플레이트 코드 많음

**추천 사용처**:
- 마이크로서비스 아키텍처
- API Gateway
- 표준 호환성이 중요한 경우
- 최소한의 의존성을 원할 때

---

### 3. Echo - 균형 & 기능 풍부

```bash
cd Echo && go run main.go
```

**장점**:
- 🎯 자동 바인딩
- 📦 가장 많은 내장 미들웨어 (CORS, JWT, Gzip 등)
- 📖 최고 수준의 문서
- 🔌 WebSocket 내장 지원
- ⚡ HTTP/2 기본 지원
- 🌐 서브도메인 라우팅

**단점**:
- ❌ Gin보다 약간 느림
- ❌ 표준 http 인터페이스가 아님

**추천 사용처**:
- 엔터프라이즈 애플리케이션
- 실시간 애플리케이션 (WebSocket)
- 풍부한 미들웨어가 필요한 경우
- 좋은 문서가 중요한 경우

---

### 4. Beego - 풀스택 MVC

```bash
cd Beego && go run main.go
```

**장점**:
- 🏗️ 완전한 MVC 프레임워크
- 💾 ORM 내장 (Bee ORM)
- 📚 자동 API 문서 생성 (Swagger)
- 🔐 세션 관리 내장
- 🌍 i18n 지원
- ⚙️ 캐싱, 스케줄링 등 모든 기능 포함

**단점**:
- 🐌 다른 프레임워크보다 느림 (40,000 req/s)
- 🏋️ 더 무거움
- 📚 학습 곡선이 가파름

**추천 사용처**:
- 대규모 엔터프라이즈 애플리케이션
- 풀스택 웹 애플리케이션
- ORM이 필요한 경우
- 자동 문서화가 필요한 경우
- 중국 시장

## 🎯 선택 가이드

### 질문으로 선택하기

**Q: 가장 빠른 개발 속도가 필요한가?**
→ **Gin** (자동 바인딩 + 큰 커뮤니티)

**Q: 표준 라이브러리와의 호환성이 중요한가?**
→ **Chi** (100% 표준 호환)

**Q: 가장 많은 기능이 필요한가?**
→ **Echo** (내장 미들웨어 최다) 또는 **Beego** (풀스택)

**Q: 마이크로서비스를 만드는가?**
→ **Chi** (경량) 또는 **Gin** (성능)

**Q: ORM이 필요한가?**
→ **Beego** (유일하게 ORM 내장)

**Q: WebSocket이 필요한가?**
→ **Echo** (내장 지원)

**Q: 가장 적은 의존성을 원하는가?**
→ **Chi** (의존성 1개)

**Q: 엔터프라이즈 애플리케이션인가?**
→ **Beego** (풀스택 MVC) 또는 **Echo** (균형)

## 🚀 빠른 시작

각 프레임워크를 테스트하려면:

```bash
# Gin
cd Gin && go run main.go
# 다른 터미널에서
./test.sh

# Chi
cd Chi && go run main.go
./test.sh

# Echo
cd Echo && go run main.go
./test.sh

# Beego
cd Beego && go run main.go
./test.sh
```

모든 서버는 `http://localhost:8080`에서 실행되며, 동일한 API를 제공합니다.

## 📝 API 엔드포인트

모든 프레임워크에서 동일한 API를 제공합니다:

- `GET /health` - 헬스 체크
- `GET /api/v1/items` - 아이템 목록
- `GET /api/v1/items/:id` - 특정 아이템
- `POST /api/v1/items` - 아이템 생성
- `PUT /api/v1/items/:id` - 아이템 수정
- `DELETE /api/v1/items/:id` - 아이템 삭제
